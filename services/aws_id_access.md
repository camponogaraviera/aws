# [AWS IAM](https://aws.amazon.com/iam/)

AWS Identity and Access Management (IAM) is used to define user access/authorization to different AWS services.

---

# [Amazon Cognito](https://aws.amazon.com/pm/cognito)

AWS Cognito is used to define customer identity and access management (CIAM) such as user authentication (sign in, sign up), role-based access/authorization to AWS services (S3, DynamoDB, AWS Lambda..), and B2B identities. Cognito console creates IAM roles by default to provide access to Mobile Analytics and Amazon Cognito Sync.

# Authorization vs Authentication

- Authorization: determines if the user has the right permission to perform operations that modify data. Anonymous users can only read data.
- Authentication: is the process of identifying if the user is who he claims he is.
